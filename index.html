Interactive experiment exploring ESP32-CAM, hardware debouncing, and secure cloud API classification.

🎯 Project Aim

This section outlines the primary objective of the experiment and the specific physical components required. It provides the foundation for understanding what is being built and tested. To construct an Internet of Things (IoT) device using an ESP32-CAM microcontroller and an OV2640 camera sensor to capture image data on a physical switch trigger and securely transmit it to a remote computer/cloud API for real-time automated waste classification.

🧠 Core Scientific Principles

Understanding the underlying mechanics is crucial. This section breaks down the three main theoretical concepts applied in this experiment: sensor interfacing, signal filtering, and network protocols.

📷

1. MCU & Sensor Interface

The ESP32-CAM connects to the OV2640 via a parallel interface. High-speed signals synchronize raw pixel capture.

PCLK HREF VSYNC
⏱️

2. Hardware Debouncing

Mechanical switches bounce. A software filter prevents multiple triggers for a single press.

Δt = tcurrent - tlast_trigger > tdebounce

Where tdebounce = 500 ms
🌐

3. Secure Network Protocol

Data is sent via TLS/SSL (Port 443) using HTTP POST multipart/form-data. Payload sizing is strict.

Ltotal = Lheader + Lbinary + Ltail

📋 Experiment Conclusions

Review the final inferences drawn from the trials, critical safety precautions for the hardware, and potential sources of error that could impact the system's reliability.

Inference

  • Device successfully connected to the 2.4 GHz wireless access point.
  • Hardware debouncing logic accurately prevented duplicate transmissions on button press.
  • Raw analog signals were successfully converted to binary payload and transferred to the cloud API.

Precautions

  • Power Supply Limits: Maintain stable 5V (2A min) to prevent brownout resets during transmission.
  • Polarity Check: Double-check FTDI serial connections to prevent instant chip damage.
  • ESD: Do not touch the camera sensor surface directly to prevent electrostatic damage.

Sources of Error

  • Network Latency: Fluctuations in Wi-Fi strength can cause transmission drops or timeouts.
  • Ambient Lighting: Insufficient light causes high sensor noise, directly reducing image classification accuracy.
